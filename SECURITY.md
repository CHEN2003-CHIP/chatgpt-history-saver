# Security Policy

## Reporting a vulnerability

Please do not open public GitHub issues for suspected security vulnerabilities.

Instead, report them privately using the support contact you publish for this project. If you do not yet have a dedicated security mailbox, create one before publishing the repository widely.

When reporting, please include:

- A short description of the issue
- Steps to reproduce it
- The affected version
- Any proof-of-concept details that help confirm the problem

## Scope

This project is a browser extension that reads visible ChatGPT conversation content locally for export. Security-sensitive areas include:

- Content script injection behavior
- Export file generation
- Temporary storage of export payloads
- Any future permissions or network behavior
