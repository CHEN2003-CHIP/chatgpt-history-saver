# GitHub Launch Checklist

Use this checklist after pushing the repo to GitHub to improve discoverability and increase the chance that people star it.

## Repository settings

- Add a clear repository description
- Add the project website URL if available
- Upload `marketing/large-promotional-tile-1400x560.png` as the social preview image
- Pin the repository on your profile
- Enable issues and discussions if you want community feedback

## Topics to add

Suggested topics:
- chatgpt
- browser-extension
- edge-extension
- chromium-extension
- html-export
- pdf-export
- productivity
- archive
- conversation-export

## README polish

- Keep screenshots near the top
- Show the value proposition in the first 3-5 lines
- Explain what makes this better than copy-paste
- Add a short demo GIF later if possible

## Social proof and visibility

- Create a first GitHub release
- Write a short launch post on X, Reddit, or Product Hunt
- Share a before/after example of exported conversations
- Add the repository to your profile README if you have one

## Release hygiene

- Tag the first stable version
- Attach the packaged ZIP to a GitHub release if useful
- Keep the changelog updated
